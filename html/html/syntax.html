HTML syntax

HTML has a specific syntax used for marking up text documents, this affects how the text is represented on the screen and its structure.

Elements and tags

Tags are often used in languages for many reasons, a large one is the ease of which a tag-based document can be represented in a data structure; as prior seen, the DOM can be represented as a tree.

Tags in HTML have three forms that are used to represent elements, all use angle brackets, <>, to represent elements:

  • Opening tag
  • Closing tag
  • Self-closing tag

An opening tag (tag opener) is used to declare an element, the following tag is used to declare a block-level paragraph element:

<p>

To complete a HTML element, a closing tag (tag closer) must be supplied after the opening tag to create a complete element:

HTML<p></p>

A HTML element can represent some semantic or non-semantic meaning in the HTML document, they are used for structuring content. Block-level elements, like the p element, take up the whole width of a document for example and are semantic and presentational - they indicate a paragraph of text and browsers know how to render this.

The difference between a tag opener and closer is in the forward slash before the text between the angle brackets, this signals that the element closes here. The content to display to the user is then placed between the tag opener and tag closer:

HTML<p>Hello world</p>

It is important to know that HTML elements can be nested inside each other, this is how we get nodes on a tree. It is extremely common and key to creating powerful, and semantic, designs:

HTML<p>
    The <strong>nested element</strong> in this example is bolded using a <strong>strong</strong> tag to not just visually signify 
    its importance, but also to screen-readers and other hardware and/or software for disabled users.
</p>

XML - Extensible Markup Language - is a language that uses tags, just like HTML; the difference being that XML is for mainly for storing and communicating data whereas HTML concerns the layout of a hypertext document.

Self-closing tags are a special kind of tag that only have an opening tag, with a closing forward slash at the end of the element if writing code compliant to the XHTML standard alongside HTML5. <img /> elements are a prime example of this:

HTML<img src="url" />

Attributes

An attribute is used to provide additional information about an HTML element. The key points to note about attributes are:

  • Attributes provide additional information
  • Attributes can be used on any HTML elements, although some elements are restricted in what attributes they can use
  • Attributes are always specified in the opening tag of an element, self-closing tags included
Attributes are commonly supplied to HTML elements, a prime example is using the img element to load images into our HTML documents: 
HTML<img src="url" />
  • The src attribute is a key=value pair where src is the key and the value is within the quotation marks.

The src attribute of an image is capable of pointing to images on the web, and images stored locally on your computer. Try this by creating a an image element like the following:

HTML<img src="c://path/to/image.png" />

If the image is located within the same folder as the HTML document, or a nested folder, use a relative path to easily point to this instead of specifying a whole absolute path. For example, given the following folder structure:

 Root
|
 --- index.html
 --- images
 --- --- image.png

The project root contains a file, index.html, and a folder, images. Inside the images folder is an image, image.png. To access this from an image tag inside index.html, we would use the following relative path:

HTML<img src="/images/image.png" />
