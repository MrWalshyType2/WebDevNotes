Forms

HTML has a form element which can be used for creating forms to collect data from users, and then submit to a server for processing of said data.

  • Please note that this module covers only the core information for building a form, submitting form data and handling responses requires a server or programmatic control of the form using JavaScript.

HTML Forms

To create a form, the form element must first be specified as a container for holding form elements:

<form method="" action="">
    <!-- form elements in here -->
</form>

The form element has two mandatory attributes for submitting data to a server using the traditional client-server model:

  • method: The HTTP request method in use (GET or POST)
    • GET: Used for reading data from a server, data may be sent in a query parameter for example: google.com?q=cats
    • POST: Used for sending data to a server (sent in the request body)
  • action: The URL or script to send data to for processing

After defining the container for the form element, we can then create input elements inside the form for users to enter data into, for example:

<form action="" method="POST">
    <!-- input elements
            - various types built into HTML5
            - some input types come with validation enabled, such as for emails
            - placeholder attribute signifies placeholder data
            - value attribute specifies content inside in the input field
            - required attribute means the form will not submit unless the value is present
            - name identifies what the input element belongs to
            - id is a unique identifier
    -->
    Name:
    <input type="text" name="name" placeholder="Bob Smith" required />
</form>

The above example would render as follows:

Name:

The input is an inline element and by default is used for text input, we can also specify this using the type attribute as seen above. Other attributes we may wish to use include:

  • name: Specifies the name of the input field, this is not seen by the user but is submitted alongside the input fields value as a key=value pair. If omitted, the input is not submitted.
  • placeholder: Signifies a placeholder value for the input field that disappears when the user starts inputting data
  • value: Used to specify default data that is already present in the input field
  • required: Indicates the input field as mandatory, prevents the form from being submitted unless data is present. This attribute does not take a value.
  • readonly: The field is still visible but is unusable - the data will still be sent to the server and users can still click/tab onto the field. This attribute does not take a value.
  • disabled: The input field is still visible but is unusable, unclickable and will not be sent to the server. This attribute does not take a value.
  • size: Specify the width, in characters, of the input field
  • maxlength: Sets the maximum number of allowed characters in an input field
  • min and max: Works only with numerical and date types, used to specify the minimum and maximum values for an input field
  • pattern: Allows the specification of a regular expression for validating form data against

Form input types

There are various types of inputs that can be supplied in a HTML form. Prior to HTML 5, there was 10 basic input element types that could be supplied to the type attribute (https://www.w3.org/TR/html401/interact/forms.html):

  • text: A text input field
  • password: A text input field where the inputted characters are displayed as asterisks
  • checkbox: A checkbox input element
  • radio: A radio button input element.
  • image: Creates a graphical submit button where the src attribute specifies the image to use, ensure the alt attribute is specified also. These are supplied to the input element itself.
  • file: Creates a file upload option
  • hidden: Creates a hidden input field, often used for holding the identity or some reference to a record to be edited in a database.
  • button: Creates a push button input field, use the value attribute to specify the buttons name.
  • submit: Creates a submit button that submits the form to the specified form action using the specified form method
  • reset: Creates a reset button that resets the values in the form
HTML5 introduced a variety of new form input types to make forms easier to create and use, these types include:

The email attribute

The email input type is specified as follows:

<input type="email" name="email" />

When rendered in a form, it looks as follows:

Try pressing the submit button with no email present, or a malformed email, and client-side validation will be applied to the email. This is not an exhaustive security measure though, it is required that the server also checks the formatting of an email.

The search attribute

The search input type is used for creating search boxes on web pages and web applications:

<input type="search" name="q" />

When rendered in a form, it looks as follows:

Search:

Search input fields often default styling and behaviours applied to them by the browser (not always), including rounded borders, default placeholder text, a button for clearing text from the field, etc...

The tel attribute

The tel attribute is used for telephone numbers, on mobile devices it will cause them to display a numeric keypad rather than the alphabetical keyboard:

<input type="tel" name="tel" />

When rendered in a form, it looks as follows:

Telephone number:

This field does not apply client-side validation by default, meaning it can also accept letters as input. It also doesn't need to be used solely for telephone numbers, it can be useful where any numerical input is needed as it opens the numerical keypad by default on mobile devices (there is a separate number input type which is better suited though).

The number attribute

The number attribute is used for numbers, on mobile devices it will cause them to display a numeric keypad rather than the alphabetical keyboard. A minimum and maximum value can be specified using the min and max attributes respectively: 

<input type="number" name="number" min="18" max="130" />

When rendered in a form, it looks as follows:

Age:

This field type has a basic client-side validation where it only allows integers (whole numbers). To change this, set the step attribute equal to any to allow floating point numbers (decimals). For this input type, browsers will also usually include controls for incrementing and decrementing the input number by the specified step value.

Other useful types

Alongside those mentioned, HTML5 introduced the following useful types for controlling form inputs:
  • url: Used for specifying a URL, provides client-side validation that indicates a URL is malformed if no protocol is specified such as http: or https:.
  • range: Used for a specifying a slider control on a form, used for picking a value within a range but suffers from imprecision in its control. Like the number type, a min, max and step attribute can be supplied. The current value of the slider is not displayed by default and requires JavaScript to do so.
  • Date and time pickers: Various types exist for specifying dates and time, such as:
    • datetime-local: Displays a widget for picking a date and time.
    • month: Displays a widget for picking a month and year.
    • time: Displays a widget for selecting a time.
    • week: Displays a widget for selecting a week number and year.
  • color: Displays a colour picker widget for selecting a specific colour.

Simple form example

The following example demonstrates using a form to submit search text to Google, Google's server will respond with a redirect which by default is then followed by the browser:

<form action="https://www.google.com/search" method="get">
    Google search:
    <input type="search" name="q" />
    <button type="submit">Search</button>
</form>
