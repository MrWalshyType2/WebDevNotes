Tables

Tables are used in HTML to represent tabular data (data in rows and columns).

Creating a basic table

A HTML table is created with the table element, the rows and columns are nested inside the element. The following example demonstrates some simple markup for a table with three rows (one header row) and three columns: 

<table>
    <tr>
        <th>Username</th>
        <th>Email</th>
        <th>Date of birth</th>
    </tr>
    <tr>
        <td>user123</td>
        <td>user123@mail.com</td>
        <td><time datetime="1987-04-17">17/04/1987</time></td>
    </tr>
    <tr>
        <td>user356</td>
        <td>user356@mail.com</td>
        <td><time datetime="1994-09-23">23/09/1994</time></td>
    </tr>
</table>
  • Each row in the table is represented by the tr element, the data which represents the columns is nested inside of the tr elements.
  • The th element represents a table heading, this table has three headings on the first row which represent the headings for three columns.
  • The td element represents table data, each piece of data conforms to the columns defined prior. In this case, the first piece of data in each row after the first row (the row with the headings) is the username and will be lined up in the first column.

By default, tables have minimal styling and do not expand to the width of the parent container like most block-level elements. The previous markup would look as follows by default:

Username Email Date of birth
user123 user123@mail.com 17/04/1987
user356 user356@mail.com 23/09/1994

Spanning multiple rows and columns

Sometimes, we may have data that requires a more complex layout... To handle this, the colspan and rowspan attributes can be applied to the table cells. We will also make sure to specify the scope attribute and set it appropriately, this helps signify to screenreaders what a th element is being applied to (a column (col) or a row (row)).

The following example demonstrates a heading applied to the bottom row to represent the total cost of an order:

Item Quantity Cost
6 pack apples 1 £1.70
Chocolate Orange 3 £1.00
Total £4.70

We may also want to span more than one row, the following example demonstrates this using a primary and secondary email:

Username Email Date of birth
user123 Primary: user123@mail.com 17/04/1987
Secondary: user123alt@mail.com
user356 Primary: user356@mail.com 23/09/1994
Secondary:

Adding a caption

A table can be captioned to aid accessibility and provide a visible descriptive text, the caption element is used for this:

Table of users
Username Email Date of birth

Adding structure to a table

When creating a table, it can be structured semantically using the thead (table head), tbody (table body), and tfoot (table footer) elements.

  • The thead element wraps the header of the table, usually the first row which contains the column headings.
  • The tbody element wraps the body of the table, the main data rows of the table.
  • The tfoot element wraps the footer content of the table, this will always be rendered at the bottom of the table regardless of its position.

We will add these elements to the table of items in an order from earlier, visually there won't be any change but semantically the table has much more meaning:

Item Quantity Cost
Total £4.70
6 pack apples 1 £1.70
Chocolate Orange 3 £1.00
  • As prior mentioned, the table footer appears at the bottom even though in this example it has been specified under the table head element.
