Lists

This section covers the three types of HTML list and nested lists.

In HTML documents, we can represent lists of data. In HTML, there are three types of list:

  • Unordered lists
  • Ordered lists
  • Description lists

We can also nest these lists inside of each other.

Types of list

Unordered lists

Unordered lists are used for representing unordered data, such as a list of favourite vehicles or foods. There are two key elements in an unordered list, ul and li. The ul element is the container element for the list, nested within the ul element is possibly multiple li elements. For example:

<ul>
    <li>Mazda 787B</li>
    <li>Sauber C9</li>
    <li>Spyker C8</li>
</ul>

This will render an unordered list of elements to the screen, specifically the content within the li elements which will be listed using bullet points by default in most browsers.

Ordered lists

Ordered lists are the inverse of unordered lists, these allow us to structure lists of items in an order. By default, an ordered list will be rendered using numbers. Try the following example:

<ol>
    <li>Eggs</li>
    <li>Apples</li>
    <li>Oranges</li>
</ol>

As mentioned, the default type for an ordered list is to use numbers, we can also change the type using the type attribute to one of the following:

Type Description
1 Default if no type specified, orders numerically from smallest value to largest by default, 1...n items
A Orders the list using uppercase alphabetic letters.
a Orders the list using lowercase alphabetic letters.
I Orders the list using uppercase roman numerals.
i Orders the list using lowercase roman numerals.

For example, I can order the list from before using lowercase roman numerals like so:

<ol type="i">
    <li>Eggs</li>
    <li>Apples</li>
    <li>Oranges</li>
</ol>

The starting numeral, or letter, used for indicating the order can be specified using the start attribute. If I wanted to start the ol from the roman numeral ii, I would do the following:

<ol type="i" start="2">
    <li>Eggs</li>
    <li>Apples</li>
    <li>Oranges</li>
</ol>
The start attribute only takes integer values as input, even for non-numeric list types.

We can reverse the order of the list by using specifying the reversed attribute in the ol opening tag, for example:

<ol type="i" reversed>
    <li>Eggs</li>
    <li>Apples</li>
    <li>Oranges</li>
</ol>
Some attributes are not key=value pairs, they are instead like a boolean flag. For example, if the reversed attribute is present it is true that the list is reversed and vice-versa.

Description lists

A description list is a list where each element, known as a description term, can have many associated data definitions. For example:

<dl>
    <dt>Mazda 787B</dt>
    <dd>A rotary powered LMP Car</dd>
    <dd>The rotary engine is also known as the Wankel engine</dd>
    <dd>The only rotary powered car to win the 24 hour le mans</dd>
    
    <dt>Toyota TS050</dt>
    <dd>An experimental hybrid LMP car</dd>
</dl>

First, a dl element is specified that acts as the container for the description terms - this is similar to ul and ol. Each element of the description list is then specified using dt elements (Mazda 787B and Toyota TS050 above are the two elements) followed by zero or many dd elements. Every dd element following a new dt element belongs to that description term in the list.

Nested lists

Lists, like most HTML elements, support the nesting of other elements inside them - this also means we can nest lists inside of other lists. For example, the following unordered list of categories (Favourite cars and foods) each contain a description list nested within the li element of the ul:

<ul>
    <li>Favourite cars:
        <dl>
            <dt>Mazda 787B</dt>
            <dd>Only rotary engine powered car to win the 24 hour le mans</dd>

            <dt>Nissan Skyline R34</dt>
            <dd>Revolutionised JDM and tuner street racing</dd>
        </dl>
    </li>
    <li>Favourite foods:
        <dl>
            <dt>Pizza</dt>
            <dd>Can't beat a pepperoni</dd>

            <dt>Grill cheese</dt>
            <dd>Yumm!</dd>
        </dl>
    </li>
</ul>
