Semantic elements and accessibility

Semantic elements have always been a part of HTML, but so also have non-semantic elements like the infamous div for creating divisions in your document. Historically, pre-2014 before HTML5 was released, and even still to this day, document layouts have been made using div elements. For example, a typical layout used div elements a lot with classes and id's to identify them:

<div id="navbar">
    <img class="logo" alt="#" src="#" />
    <ul>
        <li class="nav-link"><a href="#">Home</a></li>
        <li class="nav-link"><a href="#">About</a></li>
        <li class="nav-link"><a href="#">Contact</a></li>
    </ul>
</div>
<div id="header">
    <h1>The example of poor HTML</h1>
    <p>And some follow on text...</p>
</div>
<div id="main">
    <p>The main content of the page</p>
</div>
<div id="footer">
    <p>Copyright ©</p>
</div>

There are two major problems:

  1. This design is not semantic enough, div elements are for structure rather than meaning. This reduces accessibility for those that rely on screen readers to indicate different parts of the page. This could be solved in HTML4 using various ARIA attributes but is not ideal as it further pollutes the source: https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA
  2. It is hard to read and thus maintain.

The rest of this section will discuss the new semantic elements added in HTML5 designed to alleviate the troubles of div based designs.

Exploring semantic elements

HTML5 added semantic elements to the language to aid accessibility and design of documents by conveying meaning in the design. These new elements include:

  • article
  • aside
  • details
  • figcaption
  • figure
  • footer
  • header
  • main
  • mark
  • nav
  • section
  • summary
  • time

Before exploring the individual elements, lets look at what the above design looks like with semantic elements in use:

<nav>
    <img class="logo" alt="#" src="#" />
    <ul>
        <li class="nav-link"><a href="#">Home</a></li>
        <li class="nav-link"><a href="#">About</a></li>
        <li class="nav-link"><a href="#">Contact</a></li>
    </ul>
</nav>
<header>
    <h1>The example of good HTML</h1>
    <p>And some follow on text...</p>
</header>
<main>
    <p>The main content of the page</p>
</main>
<footer>
    <p>Copyright ©</p>
</footer>

One thing that is immediately noticeable is how much more readability the HTML now has, the document holds much more semantic meaning...

Headers, sections and articles

Headers

The header element is used to represent the main heading of a document, section or article.

The header element should not be used inside the main element unless inside a section or article as the main element signifies the pages main content.

The design of a header differs somewhat and may follow the example seen before where it is just a header with some lead text below the navigation bar but above the main content:

<header>
    <h1>The example of poor HTML</h1>
    <p>And some follow on text...</p>
</header>

We commonly also see search tools and navigation bars inside the main header of a page:

<header>
    <h1>Company</h1>
    <nav>
        <img class="logo" alt="#" src="#" />
        <ul>
        <li class="nav-link"><a href="#">Home</a></li>
        <li class="nav-link"><a href="#">About</a></li>
        <li class="nav-link"><a href="#">Contact</a></li>
        </ul>
    </nav>
</header>

Sections and articles

HTML5 added section elements for thematically grouping content and article elements for representing independently distributable or reusable component of the page.

  • Articles could include blog posts, forum posts, comments, widgets, etc...
  • Nested articles should be related to their parent article, such as comments on a blog post
  • Sections and articles should be identified by a heading
  • Sections cannot usually be used independently as they represent a piece of the larger whole document

Sections may contain articles and articles may contain sections, what is most important is that their use makes semantic sense. For example, a blog post might look as follows:

<article class="blog-post">
    <h2>The posts title</h2>
    <section class="content">
        <h3><u>A lead title</u></h3>
        <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nulla accumsan ipsum quis dui varius ornare. Sed nec eros sed justo volutpat fermentum. Fusce egestas malesuada euismod. Cras imperdiet congue erat nec volutpat. Sed condimentum rutrum urna vel euismod. Etiam et est tellus. Mauris quis ante nisl. Suspendisse eros felis, sodales non commodo sed, gravida a elit. Morbi tempus ipsum id faucibus placerat. Etiam eu pellentesque sem, quis vulputate mauris. Ut a pretium sem. Nullam hendrerit bibendum molestie. In hac habitasse platea dictumst. Proin venenatis tellus at dui sagittis, nec tempus justo vehicula. Donec dapibus magna non commodo consequat.</p>
    </section>
    <section class="comment-area">
        <h3><u>Comments</u></h3>
        <article class="comment">
        <h4>User 1</h4>
        <p>
            I posted this at 
            <time datetime="2022-03-31 16:00">16:00</time>
        </p>
        </article>
        <article class="comment">
        <h4>User 2</h4>
        <p>
            I posted this at 
            <time datetime="2022-03-31 16:05">16:05</time>
        </p>
        </article>
        <article class="comment">
        <h4>User 3</h4>
        <p>
            I posted this at 
            <time datetime="2022-03-31 16:10">16:10</time>
        </p>
        </article>
    </section>
</article>
The time element has also been used to demonstrate it, this increases the accessibility of reading times via a screen-reader. If this element is not used, the time will instead be read like text, i.e., one six colon one zero - which does not make much sense.

The nav element

The nav element is used to indicate an area of internal or external navigation and is designed to increase accessibility, this should be preferred over div based navigation bars. A simple accessible example is:

<nav>
    <a href="#">Index</a>
    <a href="#">About</a>
    <a href="#">Contact</a>
</nav>

figures and figcaptions

A figure element is designed to be wrapped around an image, the figcaption element is used to supply a caption to the image. For example:

<figure>
    <img src="https://via.placeholder.com/50" alt="" />
    <figcaption>This image has a caption</figcaption>
</figure>

This will render an image with a caption associated with it, this primarily aids accessibility but is useful in development as it no longer requires the use of divs and CSS styling to achieve the same result.
