Headings, paragraphs and text formatting

In HTML, we many elements with different semantic meanings. Some of the most commonly used are the elements which represent headings and paragraphs of text.

https://developer.mozilla.org/en-US/docs/Web/HTML/Element/Heading_Elements

Headings and paragraphs

In HTML, we have six sizes of heading supplied by default. These are represented by the following tags:

<h1>Largest</h1>
<h2>Second largest</h2>
<h3>Third largest</h3>
<h4>...</h4>
<h5>...</h5>
<h6>Smallest</h6>

With heading elements, they progressively get smaller. Heading elements are useful for helping section textual content, especially for screen-readers who rely on the use of semantic elements to indicate the structure of the content. For example, the following example demonstrates only using p elements to structure our content - making it inaccessible to disabled users who may not be able to distinguish the separate areas of content:

<p>Headings and paragraphs</p>

<p>Headings</p>
<p>Heading elements represent headings and have six sizes</p>

<p>Paragraphs</p>
<p>Paragraphs represent blocks of text</p>

From a presentational aspect, this is ok but nowhere near ideal - this relies on visual skills to infer the different sections of the document. To group the content more appropriately, we would use heading tags:

<h1>Headings and paragraphs</h1>

<h2>Headings</h2>
<p>Heading elements represent headings and have six sizes</p>

<h2>Paragraphs</h2>
<p>Paragraphs represent blocks of text</p>

Using heading elements will provide two major changes:

  • The presentation of the text will change to represent the default browser sizing and styling of the heading element, including text size, text font, text weight, line height, etc...
  • Heading elements create a semantic section of the document indicating that content following the heading belongs to that heading - this is very important for screen-readers. For example, the h1 element has two inner headings, h2 elements, each with their own content, providing a structure for the content.
Only one h1 element should be used per HTML document, the HTML specification doesn't state this as a rule but it generally improves accessibility for screen-readers. It is also important to ensure you use headings from largest to smallest, do not skip adding a h1 and still use h2 and h3 elements for example.

Formatting

Headings and paragraph elements contain text, this text can be formatted using certain inline elements, inline meaning that they do not cause the text to go to a new line. The following table lists some of these elements:

Element Description Example
mark Highlights the contained text visually and provides semantic meaning to screen-readers..
b Boldens the contained text (non-semantic).
i Italicises the contained text (non-semantic).
u Underlines the contained text (non-semantic).
s Puts a strike through the contained text (non-semantic).
strong Boldens the contained text and provides semantic meaning to screen-readers.
em Italisices the contained text and provides semantic meaning to screen-readers by way of emphasising the text when it is read.
abbr Abbreviates the contained text and provides semantic meaning to screen-readers.
ins Underlines the contained text and provides semantic meaning to screen-readers that the text was inserted.
del Strikes through the contained text and provides semantic meaning to screen-readers that the content is deleted.
sup Superscripts the contained text and provides semantic meaning to screen-readers.
sub Subscripts the contained text and provides semantic meaning to screen-readers.

Copy and and paste the following example into a file called index.html and open this in your browser:

  • Try changing text to see how it affects what is displayed
  • Try adding in your own inline elements for customising the presentation of your text
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Text Formatting</title>
</head>
<body>
    <h1>Text Manipulation</h1>

    <!-- Highlighting text relevant to some context (semantic) -->
    <h2>mark</h2>
    <p>Hello there <mark>Bob</mark></p>

    <!-- Bold, italic and underlined text (non-semantic) 
            - b tag bolds text
            - i tag italicises text
            - u tag underlines text
            These are visual and do not offer semantic help to screen readers.
    -->
    <h2>Non semantic bold, italic and underline</h2>
    <p>Hello <b>Bob</b>, <i>how are <u>you</u> doing</i>?</p>

    <!-- Bold & Italic (semantic) 
            - strong tag for strong reading (bold text)
            - em tag for emphasis on reading (italic text)
            These adjust how the content is read by screen readers (accessibility focused)
    -->
    <h2>Semantic bold and italic</h2>
    <p>Hello <strong>Bob</strong>, how are <em>you</em> doing?</p>

    <!-- Abbreviations (semantic)
            - abbr tag for abbreviations
            - title attribute to indicate what is being abbreviated
    -->
    <h2>Abbreviations</h2>
    <p>I am studying <abbr title="HyperText Markup Language">HTML</abbr></p>

    <!-- Inserted, Deleted or Stricken
            - ins tag for inserted text (semantic for u tag)
            - del tag for deleted text (semantic for s tag)
            - s for strike through
    -->
    <h2>Inserted, deleted or stricken</h2>
    <p><ins>This text was inserted</ins> into here.</p>
    <p><del>This text was deleted</del> from here.</p>
    <p><s>This text was striked through</s> here.</p>

    <!-- Superscript, subscript
        - sup tag for superscript
        - sub tag for subscript
    -->
    <h2>Superscript and subscript</h2>
    <p>27<sup>o</sup>C</p>
    <p>The number in base 10 is: 16<sub>10</sub></p>
</body>
</html>
